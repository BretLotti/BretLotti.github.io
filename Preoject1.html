Pushing and Pulling

Pushing in a basic sense is exactly what it sounds like. You're pushing changes you make in Visual Studio to Github. This what is updates the website itself. So when you visit it, the changes you've made in visual studio are reflected on the website. Pulling brings changes back in and in my experince has been used tp fix issues that may arise when trying to push changes through.

HTML/CSS

HTML and CSS and programming languages that are used for the creation of websites. For example, this entire website you are on right now was made using a mix of HTML and CSS. All the images, text, linking, formatting, etc are all done using these two languages. Web browsers such as Chrome, Firefox, and Edge, all read HTML files and display their contents as a webpage.

What is Git?

To put it simply. Git(Which is a Verison Control System) tracks and handles changes you make to files. Git stores changes we make to code in a repository, which allows us to then see the changes we have made. This can be very helpful when working and sharing files/code with other people. Though it is also very helpful in keeping track of our own work.
